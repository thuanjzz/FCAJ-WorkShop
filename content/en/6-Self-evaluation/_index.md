### Areas for Improvement

* **Written async communication:** Should document design decisions and blockers more proactively in GitHub Issues rather than only discussing verbally in meetings.
* **Time estimation:** Underestimated the complexity of the AWS Bedrock integration (throttling, IAM permissions, VPC networking) — need to add buffer time for cloud-specific tasks.
* **Testing discipline:** Unit tests and integration tests were added late in the project; should adopt TDD earlier in future sprints.
* **English technical writing:** Improve ability to write clear, concise technical documentation in English for international team collaboration.
