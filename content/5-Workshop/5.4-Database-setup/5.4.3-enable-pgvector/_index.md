Once the Amazon RDS PostgreSQL instance is successfully provisioned, the next step is to connect to the database and activate `pgvector` - the core extension that transforms traditional PostgreSQL into a Vector Database capable of storing AI embeddings.

Because our RDS cluster is securely placed in a **Private Subnet** (following the Zero-Trust model), it cannot be accessed directly from the Internet. Instead, this operation must be performed through an intermediary server (Bastion Host / EC2) located inside the VPC.

#### 1. Retrieve Connection Information (Endpoint)
1. Navigate to **RDS Console** → **Databases**.
2. Click on `cloudforge-db` (ensure its status has changed to *Available*).
3. In the **Connectivity & security** tab, copy the URL under the **Endpoint** section (e.g., `cloudforge-db.xxxxxx.ap-southeast-1.rds.amazonaws.com`).

#### 2. Connect and Enable the Extension
We will use an EC2 instance (Bastion Host) assigned with the `cloudforge-ecs-app-sg` Security Group to pierce through the Database's firewall. From the EC2 Terminal, execute the following commands in order:

**Install the PostgreSQL Client tool:**
```bash
sudo dnf install -y postgresql15
```

**Connect to the Database:**
```bash
psql -h <ENTER_YOUR_ENDPOINT_HERE> -U postgres -d cloudforge_db
```
*(The system will prompt you for a password; enter the master password you set during RDS initialization).*

After successfully logging into the PostgreSQL prompt (`cloudforge_db=>`), run the following SQL command to activate the extension:

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

#### 3. Verify the Installation
To confirm that the extension was installed successfully, run:

```sql
SELECT extname, extversion FROM pg_extension WHERE extname = 'vector';
```

If the results display `vector` along with its version (e.g., `0.8.1`), congratulations! Your database is now fully prepared to store high-dimensional vectors generated by Amazon Titan.
